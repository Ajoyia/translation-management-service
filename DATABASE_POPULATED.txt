## Command Options

### Basic Usage
```bash
# Populate with default 100k records
php artisan translations:populate

# Custom amount
php artisan translations:populate 50000

# Custom chunk size (default 1000)
php artisan translations:populate 100000 --chunk=2000

# Larger datasets
php artisan translations:populate 500000
php artisan translations:populate 1000000 --chunk=5000
```

### What the Command Creates

**Translation Keys**: Generated from realistic patterns
- Prefixes: button, label, message, error, success, warning, title, description, placeholder, tooltip, heading, nav, menu, form, field, action, status, notification
- Suffixes: login, logout, submit, cancel, save, delete, edit, create, update, view, search, filter, sort, export, import, download, upload, confirm, reset, back
- Format: `{prefix}_{suffix}_{unique_id}`

**Content**: Generated from templates
- "Welcome to %s"
- "Click here to %s"  
- "Please enter your %s"
- "Your %s has been saved"
- "Error: %s not found"
- And more...

**Tags**: Randomly assigned 1-3 tags per translation

## Production Considerations

### For 1M+ Records
1. **Use MySQL instead of SQLite** for better performance
2. **Enable Redis caching** for export endpoints
3. **Database sharding** by locale for horizontal scaling
4. **Read replicas** for distributed queries
5. **Elasticsearch** for advanced search capabilities

### Current Setup Supports
- ✓ Up to 500k records smoothly on SQLite
- ✓ 1M+ records with MySQL/PostgreSQL
- ✓ Multi-million with proper indexing and caching

## Cleanup

To reset the database:
```bash
# Remove all translations
php artisan migrate:fresh --seed

# This will:
# - Drop all tables
# - Recreate schema
# - Seed with initial 30 test translations
# - Keep 5 base tags
```
